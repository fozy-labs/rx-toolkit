# Migrating from v0.5.x to v0.6.0

## Overview

v0.6.0 is a major release that promotes Query v2 (`unstable_queryV2`) to the primary query API and removes Query v1 entirely.

## Step 1: Update Imports

### Before (v0.5.x)

```typescript
// v1 API (direct imports)
import { createResource, createCommand, useResourceAgent, useCommandAgent, SKIP } from '@fozy-labs/rx-toolkit';

// v2 API (namespace import)
import { unstable_queryV2 } from '@fozy-labs/rx-toolkit';
const api = unstable_queryV2.createApi({ ... });
api.createResourceV2({ ... });
api.createCommandV2({ ... });
```

### After (v0.6.0)

```typescript
// All v2 APIs are now direct imports (no namespace, no V2 suffix)
import { createApi, ReactHooksPlugin, commandLink, SKIP } from '@fozy-labs/rx-toolkit';

const api = createApi({ plugins: [new ReactHooksPlugin()] });
api.createResource({ ... });
api.createCommand({ ... });
```

## Step 2: Rename V2 Symbols

All "V2" suffixes have been removed. Use find-and-replace in your codebase:

| Before (v0.5.x) | After (v0.6.0) |
|---|---|
| `unstable_queryV2.createApi()` | `createApi()` |
| `api.createResourceV2()` | `api.createResource()` |
| `api.createCommandV2()` | `api.createCommand()` |
| `useResourceV2Agent()` | `useResourceAgent()` |
| `useCommandV2Agent()` | `useCommandAgent()` |
| `ResourceV2` | `Resource` |
| `ResourceV2Agent` | `ResourceAgent` |
| `ResourceV2CacheEntry` | `ResourceCacheEntry` |
| `CommandV2` | `Command` |
| `CommandV2Agent` | `CommandAgent` |
| `CommandV2CacheEntry` | `CommandCacheEntry` |
| `ResourceV2Ref` | `ResourceRef` |
| `IResourceV2` | `IResource` |
| `IResourceV2Agent` | `IResourceAgent` |
| `ICommandV2` | `ICommand` |
| `ICommandV2Agent` | `ICommandAgent` |
| `TResourceV2Options` | `TResourceOptions` |
| `TCommandV2Options` | `TCommandOptions` |
| ...all other V2-suffixed types | ...without V2 |

## Step 3: Migrate from v1 API

If you were using the v1 API (`createResource`, `createCommand` from the root), you need to migrate to the v2 API pattern:

### v1: createResource → v2: createApi + createResource

```typescript
// v1
import { createResource, useResourceAgent } from '@fozy-labs/rx-toolkit';

const myResource = createResource({
    key: 'my-resource',
    queryFn: async (args) => { ... },
});

// In component
const state = useResourceAgent(myResource, args);
```

```typescript
// v2 (v0.6.0)
import { createApi, ReactHooksPlugin } from '@fozy-labs/rx-toolkit';

const api = createApi({ plugins: [new ReactHooksPlugin()] });
const myResource = api.createResource({
    key: 'my-resource',
    queryFn: async (args) => { ... },
});

// In component
const state = myResource.useResourceAgent(args);
```

### v1: createCommand → v2: createApi + createCommand

```typescript
// v1
import { createCommand, useCommandAgent } from '@fozy-labs/rx-toolkit';

const myCommand = createCommand({
    queryFn: async (args) => { ... },
});

// In component
const [trigger, state] = useCommandAgent(myCommand);
```

```typescript
// v2 (v0.6.0)
import { createApi, ReactHooksPlugin, commandLink } from '@fozy-labs/rx-toolkit';

const api = createApi({ plugins: [new ReactHooksPlugin()] });
const myCommand = api.createCommand({
    queryFn: async (args) => { ... },
    link: [commandLink({ resource: myResource, invalidate: true, forwardArgs: () => undefined })],
});

// In component
const [trigger, state] = myCommand.useCommandAgent();
```

## Step 4: Remove Deprecated APIs

These v1 APIs no longer exist:
- `createOperation()` — Use `createCommand()` instead
- `useOperationAgent()` — Use `command.useCommandAgent()` instead  
- `createResourceDuplicator()` — No direct replacement in v2
- `useResourceRef()` — Use `ResourceRef` class instead
- `resetAllQueriesCache()` — No direct replacement; manage cache through API instance
