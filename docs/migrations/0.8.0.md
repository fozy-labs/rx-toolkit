# Миграция с версии 0.7.x на 0.8.0

В версии 0.8.0 удалено всё устаревшее (deprecated) публичное API сигналов. Ниже — список удалённых элементов и их актуальные замены.

## Signals

### `Computed.destroy()` / `Signal.compute(...).destroy()` → `dispose()`

Метод `destroy()` у `Computed` и у результата `Signal.compute(...)` удалён. Используйте `dispose()`.

```typescript
// Было
const sum$ = Signal.compute(() => a$() + b$());
sum$.destroy();    // ❌
// Стало
const sum$ = Signal.compute(() => a$() + b$());
sum$.dispose();    // ✅
```

> Сигналы также поддерживают `Symbol.dispose` (`using`), поэтому ручной вызов `dispose()` часто не требуется.

### `LocalState.create(options)` → `LocalSignal.state(options)`

Статическая фабрика `LocalState.create(...)` удалена. Используйте `LocalSignal.state(...)`.

```typescript
// Было
import { LocalState } from '@fozy-labs/rx-toolkit';

const volume$ = LocalState.create({ key: 'user-volume', defaultValue: 1 });   // ❌

// Стало
import { LocalSignal } from '@fozy-labs/rx-toolkit';

const volume$ = LocalSignal.state({ key: 'user-volume', defaultValue: 1 });   // ✅
```

> Класс `LocalState` по-прежнему экспортируется и может использоваться напрямую через конструктор (`new LocalState(options)`); удалена только устаревшая статическая фабрика `LocalState.create(...)`. Сигнатура опций (`LocalStateOptions`) и поведение `LocalSignal.state(...)` полностью идентичны прежнему `LocalState.create(...)`.

## Типы

Удалены устаревшие типы сигналов. Используйте актуальную иерархию `ReadonlySignal` / `DisposableSignal` / `StateSignal` / `LocalStateSignal`.

| Удалённый тип            | Актуальная замена                                              |
| ------------------------ | ------------------------------------------------------------- |
| `ReadableSignalLike<T>`  | `ReadonlySignal<T>`                                            |
| `ReadableSignalFnLike<T>`| `ReadonlySignal<T>`                                            |
| `WriteableSignalLike<T>` | `StateSignal<T>` (или `LocalStateSignal<T>` для local-сигнала)|
| `ClearableSignalLike<T>` | `LocalStateSignal<T>` (содержит `clear()`)                    |
| `StatefulSignalFn<T>`    | `LocalStateSignal<T>` (форма `LocalSignal.state(...)`)        |
| `SignalFn<T>`            | `StateSignal<T>`                                               |
| `ComputeFn<T>`           | `DisposableSignal<T>`                                          |

```typescript
// Было
import type { StatefulSignalFn, SignalFn, ComputeFn } from '@fozy-labs/rx-toolkit';

let local: StatefulSignalFn<number>;   // ❌
let state: SignalFn<number>;           // ❌
let computed: ComputeFn<number>;       // ❌

// Стало
import type { LocalStateSignal, StateSignal, DisposableSignal } from '@fozy-labs/rx-toolkit';

let local: LocalStateSignal<number>;   // ✅
let state: StateSignal<number>;        // ✅
let computed: DisposableSignal<number>;// ✅
```

### Новый тип `LocalStateSignal<T>`

Возврат `LocalSignal.state(...)` теперь типизирован как `LocalStateSignal<T>` — это `ReadonlySignal<T>` плюс методы записи и сброса:

```typescript
interface LocalStateSignal<T> extends ReadonlySignal<T> {
    set(value: T, actionName?: string): void;
    update(updater: (value: T) => T, actionName?: string): void;
    clear(): void;
}
```

> Рантайм-форма local-сигнала не изменилась: как и раньше, у него нет `dispose()` (в отличие от `DisposableSignal` / `StateSignal`).
