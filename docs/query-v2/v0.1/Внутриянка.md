## Как getEntry$ смотрит на пустой кеш и реагирует на `api.resetAll()`

Примерный код:
```ts
// В чем идея:
// - Реактивно наблюдать на наличие записи в кеше, не создавая ее, следую внутрениим ограничениям реализации ресурса и кеша.
// Уточннение: в реальном коде должен быть еще и `doInitiate`
getEntry$(args) {
    // Граязный binded)))
    let binded = null;
    
    // Используем сигнал, тк он может самостоятельно подписываться/отписываться
    const s = Signal.compute(() => {
        const status = this._status$();
        
        if (this._status$() === 'idle') {
            binded = null;
            return null;
        }

        if (binded) {
            return binded;
        }
        
        const entry = this._lastEntry$();

        if (entry.isMyArgs(args)) {
            binded = entry;
            return entry;
        }

        return null;
    });
}
```

### Скрытие статусы русурса

Не путать со статусами состояния кеша русурса.

- `idle` - создано, но ни одного запроса не поступило или был вызван `api.resetAll()`, и все ресурсы были удалены
- `ready` - был запрошен хотя бы один ресурс

(при необходимости можно разбить `idle` на `idle` и `reset`, но пока не вижу в этом необходимости)

### `_lastEntry$()`

Сигнал, хранящий последнюю запрошенную запись. 
Он может быть `null`, если не было ни одного запроса или был вызван `api.resetAll()`.
Важно, что он не должен хранить все записи, а только последнюю.

## Определение нарушения консистентности

- `success` и `refresh` машины могуть содержать патчи,
- при вызове `abort` может случиться так, что `immer` не сможет корректно приминить обратные патчи (выбросит ошибку) - это состояние мы называем нарушением консистентности,
- при нарушении консистентности, мы должны инвалидировать ресурс,
- в процесс инвалидации, остаются последние валидные пропатченные данные


## Сильная типизация

Места где ОДНОЗНАЧНО должна быть сильная типизация:
1) для агента, не нужно указывать `undefined`, если `args` определены, как `void`
2) `getEntry` и `getEntry$` НЕ должны возвращать `null` если мы укзали `doInitiate: true` (по умолчанию `false`), тк запись кеша будет гарантированна
3) агент также корректно должен принимать SKIP в аргументах
4) Машины, тоже сильно типизированны и имеют методы, только определенные для текущего состояние
