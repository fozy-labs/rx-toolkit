## Цели (Goals)

1. **Исправить баги, ломающие потребительский код** — `useResourceRef` мемоизация, `sideEffects: false` конфликт с `enablePatches()`
2. **Добавить недостающие тесты для критических путей** — unit-тесты core-модулей query, интеграционные тесты экспортов
3. **Исправить проблемы публичного API** — опечатки (`ResourceRefInstanse`, `FrowardInfo`, `Opertation`), недостающие экспорты типов, `any`-типы
4. **Сделать `src/query/` production-ready** — покрытие тестами core-логики, валидация barrel-экспортов

## Не-цели (Non-goals)

1. **НЕ** рефакторить архитектуру query-модуля
2. **НЕ** добавлять новые фичи (retry, pagination, refetch on focus и т.д.)
3. **НЕ** оптимизировать производительность (линейный поиск в IndirectMap, bundle size)
4. **НЕ** менять поддержку React версий (react ^19.0.0)
5. **НЕ** добавлять sub-path exports
6. **НЕ** решать архитектурные вопросы (isLocked семантика, render-phase side effects)

## Ключевые решения

| # | Решение | Обоснование |
|---|---------|-------------|
| ADR-1 | Исправить опечатки + deprecated alias | Semver 0.x допускает breaking changes; alias смягчает переход |
| ADR-2 | Экспортировать query-типы | Additive change, не ломает потребителей |
| ADR-3 | Исправить `useResourceRef` через JSON-сериализацию ключа | Минимальное изменение, совместимый паттерн |
| ADR-4 | Заменить `any` на generic constraints | Улучшение type-safety без runtime-изменений |
| ADR-5 | TODO/FIXME — задокументировать, не чинить | Не входит в scope текущей валидации |
| ADR-6 | Тестировать core без React, hooks через smoke | Максимальное покрытие при минимальных затратах |
