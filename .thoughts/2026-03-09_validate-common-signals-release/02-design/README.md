# Дизайн: validate-common-signals-release

**Status**: Черновик  
**Дата**: 2026-03-09  
**Scope**: `src/common/` и `src/signals/` (без `src/query`)  
**Цель**: Спроектировать тестовую инфраструктуру, стратегию тестирования и критические фиксы для публичного релиза

## Контекст

Данный дизайн основан на результатах исследования ([01-research](../01-research/README.md)), которое выявило:
- Полное отсутствие тестов в проекте (~950 строк кода, 19 файлов, 0 тестов)
- Ряд потенциальных багов (Batcher без try/finally, deepEqual неполная, SSR-несовместимость)
- 5 deprecated API, которые нужно сохранить до v0.6.0
- Глобальные синглтоны, усложняющие тестовую изоляцию

## Цели

1. **Создать тестовую инфраструктуру** на базе Vitest с поддержкой ESM, path aliases, jsdom
2. **Покрыть тестами все файлы** в `src/common/` и `src/signals/` (целевой coverage ≥ 80%)
3. **Исправить критические баги** — только блокирующие проблемы (Batcher try/finally и аналогичные)
4. **Задокументировать некритические проблемы** — через failing/skipped тесты и TODO-комментарии

## Не-цели

- ❌ SSR-совместимость (не требуется для этого релиза)
- ❌ CI/CD интеграция
- ❌ Удаление deprecated API (отложено до v0.6.0)
- ❌ Рефакторинг циклических зависимостей common ↔ signals
- ❌ Модификация публичного API
- ❌ Тестирование `src/query/`
- ❌ 100% code coverage
