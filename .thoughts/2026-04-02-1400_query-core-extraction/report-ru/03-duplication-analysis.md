## Краткое изложение

57 строк буквально идентичного кода присутствуют в `ResourceCacheEntry` (296 строк) и `CommandCacheEntry` (249 строк). Дублирование носит **фрагментированный** характер — распределено по 6 категориям паттернов, ни один непрерывный блок не превышает 13 строк. Большинство блоков — это последовательности из 3–5 строк вида guard-resolve/reject-null для `PromiseResolver`.

## Что НЕ является дублированием

Код, который часто ошибочно считают общим, но принадлежащий исключительно одной стороне:

| Код | Владелец | Строки | Почему он уникален |
|-----|----------|-------:|---------------------|
| Проверка гидратации (hydration check) в `_fireCacheEntryAdded` | Resource `:188–193` | 6 | У Command нет SSR-гидратации; аналога не существует |
| `_inflightPromise = null` в `complete()` | Resource `:150` | 1 | Command использует `_triggerResolver` вместо этого |
| `_patchState = null` в `complete()` | Resource `:151` | 1 | Command патчит связанные Resource'ы, а не себя |
| Отклонение `_triggerResolver` в `complete()` | Command `:230–233` | 4 | Resource дедуплицирует через `_inflightPromise` |
| `getCacheEntry` в инструментах (tools) onQueryStarted | Resource `:214` | 1 | У объекта tools в Command нет эквивалентного свойства |
| Механизм проверки устаревания (stale check) | Оба | — | РАЗЛИЧАЕТСЯ: по идентичности (`!== controller`) vs по сигналу (`.aborted`) |

## Оценка извлекаемости (Extractability Assessment)

| Метрика | Значение |
|---------|----------|
| Всего идентичных строк | 57 |
| Наибольший непрерывный блок | **13 строк** — цепочка resolver'ов в `complete()` (паттерны №4–7) |
| Блоки ≤ 5 строк | 12 из 16 паттернов |
| Блоки ровно по 4 строки | 9 паттернов (все вида `PromiseResolver` guard-action-null) |
| Процент от совокупного объёма кода (545 строк) | 10,5% |

**Профиль фрагментации**: 57 строк разбиваются на **16 отдельных блоков**, в среднем по 3,6 строки каждый. Только очистка в `complete()` предлагает связный фрагмент на 13 строк для извлечения. Оставшиеся 44 строки разбросаны между `_fireCacheEntryAdded`, `_onQueryStarted`, настройкой прерывания (abort setup) и 4 отдельными обработчиками успеха/ошибки.

**Практический вывод**: общий базовый класс или примесь (mixin) должны предоставлять ~6 небольших защищённых вспомогательных методов (protected helpers), чтобы покрыть эти фрагменты. Экономия на каждый метод составляет 3–5 строк, что делает стоимость абстракции (новый класс/примесь + связывание) сопоставимой со стоимостью самого дублирования.
