## Per-ADR Verification

### ADR-1: Module Layering Strategy (4-layer)
**Status**: MINOR INCONSISTENCY
**Decision**: 4-layer strict hierarchy (`lib/ → core/ → api/ → react/`), "Plugins alongside `api/`".
**Architecture**: §2 defines 5 runtime layers — `lib/` (L1), `core/` (L2), `api/` (L3), `react/` (L4), `plugins/` (L5). Plugins depend on `core/` AND `react/`, placing them above Layer 4.
**Issue**: ADR-1 says "Plugins alongside `api/`" (Layer 3). Architecture correctly places plugins at Layer 5 because `ReactHooksPlugin` depends on `useResourceV2Agent` (react/, Layer 4). The architecture diagram `RHP --> useResAgent` structurally proves plugins cannot be at the same level as api/. ADR-1's phrasing "alongside api/" is inexact — it could mean directory-level co-location, but "4-layer" count excludes plugins as a separate layer.
**Severity**: Low — architecture is correct; ADR-1 wording is ambiguous but not blocking.
