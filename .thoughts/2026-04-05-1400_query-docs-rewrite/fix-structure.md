# Fix: Нелогичная структура architecture.md

## Проблема

Текущий порядок секций:

1. Заголовок + одна строка отсылки к README
2. **Диаграмма компонентов** — Mermaid graph (статическая структура)
3. **Аннотации** — плоский список пояснений к нодам диаграммы
4. **Поток данных** — Mermaid sequence diagram (динамика)
5. **Глоссарий** — таблица терминов

Проблемы:

- **Нет вводного контекста.** Читатель сразу видит диаграмму без объяснения, что перед ним и зачем.
- **«Аннотации» — вагоннoe имя.** Звучит как сноски, а не как полноценная секция. Содержимое — описание компонентов, но подано плоским списком без группировки по слоям, хотя в диаграмме слои явно выделены (api, plugins, react, cache, core).
- **Нет перехода от статики к динамике.** После «Аннотаций» внезапно появляется sequence-диаграмма без объяснения, что теперь мы смотрим на runtime-поведение.
- Глоссарий на месте (справочник в конце) — это корректно.

```
  └─ Mermaid graph (без изменений).

## Компоненты по слоям
  └─ Переименовать «Аннотации» → «Компоненты по слоям».
     Разбить плоский список на подсекции по subgraph-ам диаграммы:

     ### Публичный API
       - createApi

     (правило №4 из common-mistakes.md).

## Поток данных
  └─ Одно предложение-подводка: "Ниже — типичный сценарий:

     Mermaid sequence diagram (без изменений).

  └─ Таблица терминов (без изменений, остаётся в конце как справочник).
```

## Принцип

Итоговый порядок следует top-down:

1. **Что это** (введение)
2. **Из чего состоит** (диаграмма — обзор)
3. **Детали каждого компонента** (по слоям — углубление)
4. **Как это работает в runtime** (поток данных — динамика)
5. **Справочник терминов** (глоссарий — reference)
